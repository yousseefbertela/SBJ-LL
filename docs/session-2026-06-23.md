## 1. How we PUSH (deploy)

Shopify CLI, always to the live theme with `--only` so we never clobber other files:

```powershell
$env:SHOPIFY_CLI_THEME_TOKEN="shptka_947635f12671d297209996e9ade7de7d"
shopify theme push --only "snippets/sbj-card-css.liquid" --only "layout/theme.liquid" `
  --store=sbjuve.myshopify.com --theme=141521125475 --allow-live
```

- Multiple files = repeat `--only "path"`.
- `--allow-live` is required (this is the live theme).
- **Pull EDITOR-OWNED files before editing them** (the theme editor can change them; pushing a stale copy wipes the merchant's media/settings):

```powershell
shopify theme pull --only "templates/collection.json" --store=sbjuve.myshopify.com --theme=141521125475 --nodelete
# note: pull does NOT accept --allow-live
```

**Editor-owned (pull-first, never blind-push):**
`templates/*.json` (index, collection, product…), `sections/*-group.json` (header-group, footer-group), `config/settings_data.json`.

**Catalog/data changes** (collections, products, template suffix) go through the **Shopify MCP**, not the CLI:
`create-collection`, `create-product`, `add-to-collection`, `graphql_mutation` (e.g. `collectionUpdate` for `templateSuffix`).

## 2. How we WORK (the loop + rules)

1. **Edit → push → verify → iterate.** Never assume; always screenshot.
2. **Verify with the browse tool**, not guesses:
   - `browse goto "https://sbjuve.myshopify.com/<path>?preview_theme_id=141521125475&cb=<timestamp>"`
   - `?preview_theme_id=...` + `&cb=<ts>` **bypasses Shopify's edge cache** (the live URL serves stale for minutes after a push).
   - If the browse server restarts it drops the session → re-auth: fill the password field with `mayldi` and submit.
   - Screenshot element/region, then **Read** the PNG to actually look at it. Compare to SKIMS.
3. **Native-first.** Use Horizon's native sections/blocks/settings; only hand-write CSS/Liquid for the custom parts. Keep everything editable from the theme editor where possible.
4. **Custom CSS lives in snippets rendered in `layout/theme.liquid` near `</body>`** so theme-editor saves can't revert it and it wins source-order. NEVER move CSS back into `templates/index.json` (a push there once wiped the hero photo).
5. **Scope every rule.** Homepage rules under `body:has(#sbj-home)` / `body.sbj-home`. Collection/product rules under `[data-template^="collection"]` / `[data-template^="product"]` (the `data-template` attr is on `#MainContent`).
6. **Commit after each coherent chunk**; push to GitHub `main`. Commit messages via a temp file + `git commit -F` (PowerShell here-strings mangle multi-line `-m`).

## 4. What we DID this session

### Mega menu (the big one — "speed > everything")
- **Root cause of the freeze:** Horizon paints an invisible **safety box** (`.menu-list__link[data-safety-box='true']::after`) over the active item while the pointer is moving, which blocks the next item from receiving hover until you STOP. Killed it in `sbj-menu-css`:
  `#header-component .menu-list__link[data-safety-box='true']::after { display:none !important; }`
  → panel now follows the cursor instantly in **both** directions.
- Speed token override: `--submenu-animation-speed` 360ms → **140ms** (base.css:24 is the default; our `:root` later in source wins).
- **Brown animation:** sienna `#986B58` underline slides in under the hovered/active top-level link via `.menu-list__link-title::after { transform: scaleX(0)→1 }`.
- **Pictures in the panel:** menu is `menu_style: featured_products`, but the nav items are plain `/collections/<handle>` URL links (not typed collection links), so Horizon fell back to text. Patched `snippets/mega-menu-list.liquid`:
  - force `eager_loading = true` (Horizon otherwise suppresses featured products on the hydrated re-render),
  - resolve the collection from the URL (`/collections/<handle>`) when the link isn't a `collection_link`.
  → every menu now shows 2–3 product images + text.

### Footer
- `sbj-footer-css`: newsletter heading "STAY IN THE KNOW" → **24px / lh30 / ls0.6 / weight 800**; column headings (HELP/MORE) → **16px / lh20 / ls0.4 / weight 800**; links Inter 12px/lh28.

### Homepage
- **Category row → real peeking carousel** (`sections/sbj-cat-carousel.liquid`): cards-per-view 3.5 / 2.2 / 1.3 (desktop/tablet/mobile), prev/next arrows + tiny inline JS scroller, editor settings (`cards_per_view*`, `card_gap`).
- **TRENDING cards** (`sbj-home-css` rule 10): grey eyebrow, heavy uppercase name, small price, wishlist heart, thin `#E8E5DF` dividers, dark arrows.

### Clothing collection page  (`/collections/clothing`)

- Created 8 SKIMS clothing products (see §5), tagged `clothing` + a sub-tag.
